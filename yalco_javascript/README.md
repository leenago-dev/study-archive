# JavaScript의 용도

- 이전에는 브라우저에서 `html`, `css`와 함께 실행될 코드를 담당하는 거에만 한정되어 있었는데, **node.js**가 생기고 난 후 다양한 곳에서 활용되고 있음.

# JavaScript의 특징

- JavaScript는 **인터프리터 언어** 이다.
- **인터프리터 언어**: 코드를 사용자에게 건네주기 전에, 따로 포장/가공하는 컴파일 과정을 거치지 않는다.
- i.e. Python, Ruby, JavaScript
  ➡️ 개발 과정이 빠르고 유연하지만 버그가 덜 걸러지고 실행이 조금 느리다는 단점이 있음.
- **동적 자료형**을 갖는다.
  ➡️ 숫자를 저장한 변수의 값을 문자열, 불리언, 객체나 배열, 함수로도 변환을 할 수 있음
- **객체 지향**을 지원하는 언어이다.
- _Prototype_, _First Class Object_ 등의 매력이 있음!

# 컴퓨터가 일하는 방식

1. CPU: 컴퓨터의 뇌
2. 보조기억장치 (SSD, HDD): 소프트웨어(코드)가 저장되는 공간
3. RAM: CPU가 보조기억장치에서 데이터를 꺼내서 작업하는 공간

- 변수, 상수를 비롯한 데이터들이 여기서 만들어지고 사용됨
- CPU가 아무리 빠르더라도 RAM이 작으면 많은 일을 순조롭게 해내기 어려움.
